## What we designed

**Calendar configuration system.** Spec written: `docs/superpowers/specs/2026-03-23-calendar-config-design.md`

Key decisions:
- Store calendar whitelist + tags as JSON in existing settings table (no new tables)
- `briefing-prep.py` pushes calendar list to jimbo-api; UI reads it for display
- New `--whitelist` flag on `calendar-helper.py` reads config from jimbo-api
- New admin page at `/app/jimbo/calendar` with checkboxes and tag dropdowns
- Tags are freeform strings: "options" = lower confidence, "airbnb" = booking context
- Fallback to `--primary-only` if config is empty (safe rollout)
- Pattern extensible to future config pages (email blacklist, model picker, etc.)
