## The core insight: too much in one session

Marvin's key observation: "Are we trying to do too much in one session?"

The sift-digest skill (239 lines) + daily-briefing skill (167 lines) = 400+ lines asking the LLM to do 15 distinct things: session detection, email fetch, context reading, sub-agent spawning, worker output review, worker rating, calendar reading, vault task selection, 8-section composition, surprise game, experiment logging, activity logging, recommendation logging, cost tracking, tasks triage announcement.

Jimbo reliably does 4-5 of these. The rest get dropped or hallucinated. The ones he drops (logging, workers) are the monitoring backbone — so failures are invisible.

Every prompt improvement we make (add examples, add context files, add logging steps) makes the prompt longer and reliable execution less likely. We're in a loop where fixes create new problems.

## Architecture discussion

Three options evaluated:

**Option A: Cron-orchestrated, LLM-composed.** Move worker orchestration to cron/Python. Jimbo's skill becomes 30 lines: read pre-computed results, compose briefing. Fixes plumbing and monitoring but doesn't address the quality ceiling (model still Sonnet-tier for editorial judgment).

**Option B: Fully scripted, LLM for prose only.** Python does everything deterministic. Jimbo adds personality to a near-finished template. Rejected — kills the creative cross-referencing (Gresham College moment) that makes Jimbo valuable. Wrong direction.

**Option C: Local Opus via Max plan.** Marvin has Claude Max (Opus 4.6 access) and a 24GB MacBook. Claude Code CLI supports non-interactive mode. Launchd job pulls pre-computed data from VPS, Opus does the thinking (editorial judgment, day planning, cross-referencing), pushes analysis back to VPS, Jimbo delivers via Telegram. Zero marginal cost. Laptop-dependent — needs fallback for when Mac is asleep.

**Decision: A as foundation, C as bonus layer.**

```
VPS (always on, reliable):
  Cron: email fetch, calendar, vault scoring, workers
  briefing-prep.py: assembles into briefing-input.json
  Jimbo: delivers via Telegram, handles conversation
  Fallback: composes from raw data if no Opus analysis

MacBook (powerful, intermittent):
  launchd ~06:30: pull briefing-input.json from VPS
  Claude Code CLI (Opus): editorial analysis, day planning
  Push briefing-analysis.json back to VPS
```

Opus does the thinking, Jimbo does the talking. Graceful degradation if Mac is unavailable.
