## The finding

**The Opus pipeline has never worked.** Marvin said: "I could tell immediately it wasn't Opus, and I don't understand why — I thought we had all the parts necessary to be wired up."

Investigation found the root cause in `scripts/opus-briefing.sh` line 17:

```bash
INPUT=$(ssh jimbo 'cat /workspace/briefing-input.json' 2>/dev/null) || exit 0
```

`/workspace/` is the **inside-the-sandbox** path. The script SSHes to the VPS host, where the file lives at `/home/openclaw/.openclaw/workspace/briefing-input.json`. The file doesn't exist at `/workspace/` on the host, so the command fails, and `|| exit 0` silently swallows the error.

Same bug on line 51 (the push path). Both launchd jobs (morning 06:35, afternoon 14:35) have been firing daily since ~March 5, producing empty log files and doing nothing. Ten days of silent failure.

**Fix applied:** Both paths corrected to `/home/openclaw/.openclaw/workspace/`. Verified the corrected path returns data.

## Why it was invisible

The script was deliberately written to exit silently on any failure (`|| exit 0` on every line) — the design intent was graceful degradation. But this meant:
- Log files always empty (no errors to see)
- No Telegram alert (script doesn't use alert.py)
- No experiment tracker entry (script exits before reaching Opus)
- Jimbo falls back to self-compose and carries on
- Dashboard shows nothing wrong

The only signal was the quality of the briefing itself — which is exactly what Marvin noticed.

## Patterns

- **Silent failures are the worst failures.** This is a variant of the session 4 visibility gap. The `|| exit 0` pattern optimised for resilience over observability. A single `echo "ERROR: file not found" >&2` would have caught this on day one.
- **Path confusion between sandbox and host is a recurring trap.** `/workspace/` means different things depending on context (inside Docker vs SSH to host). This has bitten us before with the sandbox security flags in the VPS stability fix.
- **Marvin's instinct is reliable data.** "I could tell immediately it wasn't Opus" — he spotted the quality gap before any diagnostic.
