## What we learned

### 1. Wrong skill invoked

Jimbo picked `morning-summary` instead of `daily-briefing`. The morning-summary skill is a 5-7 line standup designed to run as an isolated cron job at 07:30. It calls `/api/health` — which returned 401 (API key changed). It then gave up and narrated why it couldn't do anything.

The `daily-briefing` skill is the one that reads `briefing-input.json` and delivers the full briefing. Two skills that sound similar, invoked differently by cron vs model choice. If the cron trigger pointed at the wrong skill, or if the model chose `morning-summary` because its description sounded closer to what it thought was being asked, that's a routing problem — not a model intelligence problem.

### 2. SOUL.md contradicts the briefing skill

Session 17 rewrote `daily-briefing/SKILL.md` to be a reporter: "Report what you know, not what Marvin should do." But `SOUL.md` (injected every turn) still says:

> "**Day plan** — propose 3-5 activities for free gaps. End with 'Anything you'd swap or skip?' This is the most important part."

And:

> "Surface 2-3 with `priority >= 7`"

SOUL.md is the most authoritative file — it's injected on every turn. The skill file is only read when explicitly invoked. If SOUL.md says "propose a day plan" and the skill says "don't advise," SOUL.md wins. The session 17 rewrite only updated the skill, not the brain file.

### 3. Auth failures across the board

Three pipeline components returned 401 today: `email_insights`, `dispatch`, and the health API. The API key was rotated (Marvin provided a new key and URL this session: `jimbo.fourfoldmedia.uk` with a new longer key). The VPS scripts haven't been updated yet. This means:
- `health-helper.py` can't check system status (the first step of both briefing skills)
- `morning-summary` skill's only data source (health API) returned nothing
- `email_insights` and `dispatch` data missing from briefing-input.json

### 4. Priority scale mismatch

Marvin has moved to P0-P3 but the entire codebase uses 1-10. The task scoring that FAILED at 04:30 may be related — the scorer and the vault are speaking different languages. The same 5 priority 9-10 tasks have been surfacing for weeks because the scoring rubric hasn't changed.

### 5. Reasoning leak — 3rd consecutive session

Heartbeat polls dumped internal reasoning to Telegram again. Output discipline rules exist in HEARTBEAT.md and SOUL.md. The root cause needs investigation — is it the model, or are the rules positioned where they get lost in the token window? HEARTBEAT.md puts the output discipline section at lines 5-14 (top of file, good placement). SOUL.md puts output rules at line 107 (bottom of file — may get truncated or deprioritised by the model).

### 6. Surprise game fabrication

The surprise skill ran and produced a coherent-sounding output — but every data point was invented. It referenced vault task files that don't exist, email gems that weren't in the pipeline, and interests that don't match the context API. This suggests the skill ran without successfully reading briefing-input.json or the vault, and the model filled in from training data rather than admitting it had no data.
