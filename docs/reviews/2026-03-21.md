## The insight: vault as shared task system

The vault (1,633 active items, velocity 0) isn't just an old backlog to clear. It should be the **shared state** between Marvin and Jimbo. The loop:

```
Jimbo spots something actionable (email, calendar, vault connection)
  → creates a vault task with deadline
  → nudges about it (rate-limited, not 10 times)
  → Marvin says "I'll take this" or "you do it" or dismisses
  → task gets done (by either party)
  → marked done, nagging stops, velocity goes up
  → vault backlog actually shrinks
```

Right now we have the first arrow (spots things) and a broken version of the third (nudges without rate-limiting or awareness). The create, assign, track, and close steps don't exist.

This reframes the maturity ladder from session 9. Between step 2 (heartbeat fires) and step 3 (source data quality), there's a missing step: **vault as shared task system**.
