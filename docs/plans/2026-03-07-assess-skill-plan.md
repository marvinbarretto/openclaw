### Task 3: Build the /assess skill

**Files:**
- Create: `.claude/commands/assess.md`

**Step 1: Write the skill file**

The skill file goes in `.claude/commands/assess.md`. It should:

1. Accept `$ARGUMENTS` as either a URL or pasted text
2. Load context:
   - Run `curl` via Bash tool to fetch Priorities, Interests, Goals from `https://167.99.206.214/api/context/files/{slug}` with `X-API-Key` header. The API key should be read from env var `JIMBO_API_KEY`.
   - If API fails, read `context/CONTEXT-BACKUP.md` and warn Marvin
   - Read `context/TASTE.md` and `context/PREFERENCES.md` via Read tool
3. Load content:
   - If input looks like a URL (starts with http/https), use WebFetch to retrieve it
   - If WebFetch fails, tell Marvin to paste the content and wait
   - If input is text, use it directly
4. Evaluate the content against all loaded context
5. Output the structured verdict:
   - **Verdict** — one line: Skip / Bookmark / Worth exploring / Act now
   - **Relevance** — 2-3 sentences
   - **Connections** — which goals, priorities, interests it touches (by name)
   - **What you'd do with it** — concrete next step
   - **Blind spots** — what to verify, caveats

Key details for the skill prompt:
- The description/argument-hint frontmatter should match the existing command format (see `review-briefing.md` for reference)
- The jimbo-api Caddy route is `/api/context/*` — so the full URL is `https://167.99.206.214/api/context/files/priorities` (and interests, goals)
- The API requires `X-API-Key` header
- TASTE.md lives at `context/TASTE.md` relative to repo root
- PREFERENCES.md lives at `context/PREFERENCES.md` relative to repo root
- CONTEXT-BACKUP.md lives at `context/CONTEXT-BACKUP.md` relative to repo root
- The skill should be conversational after the verdict — invite discussion, not just dump and stop
- Don't be robotic about the format — the verdict sections are a framework, not a rigid template

**Step 2: Verify the skill is discoverable**

```bash
ls -la .claude/commands/assess.md
```

**Step 3: Commit**

```bash
git add .claude/commands/assess.md
git commit -m "feat: add /assess skill for evaluating links against context"
```
