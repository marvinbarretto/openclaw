# ADR-026: Cloudflare Pages Migration + Memory Auto-Commit

## Status

Accepted

## Context

Jimbo's blog deployed from the `gh-pages` branch to GitHub Pages. The `jimbo-workspace` repo had to be public for GitHub Pages (free tier). This meant memory files couldn't be safely committed — anything in the repo was publicly visible. As a result, Jimbo's memory-core files were gitignored and never backed up. We observed Jimbo forgetting things he'd built (including the blog itself) because memory files were lost between sessions.

Moving to Cloudflare Pages solves both problems: the repo can be made private (backing up memories safely) while the blog remains publicly accessible.

## Decision

1. **Migrate blog hosting from GitHub Pages to Cloudflare Pages.** Cloudflare Pages connects to the same `gh-pages` branch and serves the `blog/` directory as the site root. No build step — static HTML served directly. *(Superseded by ADR-027: Cloudflare now runs an Astro build from `blog-src/` instead of serving static HTML from `blog/`.)*

2. **New blog URL:** `https://jimbo.pages.dev/` (was `https://marvinbarretto-labs.github.io/jimbo-workspace/blog/`).

3. **Fix path prefixes.** GitHub Pages served under `/jimbo-workspace/blog/`, so internal links used that prefix. Cloudflare serves `blog/` as root, so prefixes were removed. In practice, Jimbo had already migrated most links to relative `./` paths — only one post had a hardcoded GitHub Pages URL.

4. **Make repo private** (Phase 3 — Marvin). GitHub Pages stops working; Cloudflare Pages continues.

5. **Remove `memory/` from `.gitignore`** on VPS so memory-core files are tracked in git.

6. **Add auto-commit heartbeat task.** Every heartbeat cycle (~30 min), Jimbo runs `git add -A && git commit && git push` if there are changes. This silently backs up memories, diary entries, and blog posts. The blog-publisher skill still does explicit commit+push for immediate publishing.

7. **Update all references** in this repo: SOUL.md, blog-publisher skill, rss-feed skill, CAPABILITIES.md, CLAUDE.md.

## Consequences

**What gets better:**
- Memory files are backed up automatically — Jimbo retains context across sessions
- Blog posts auto-publish within ~30 minutes of writing (heartbeat catches them)
- Repo can be private — no risk of exposing memory content, diary, or email digests
- Cloudflare Pages is faster (edge CDN) and has no repo visibility requirement

**What gets harder:**
- One more external service (Cloudflare) to manage, though it's free tier and low-maintenance
- Auto-commit means more git noise in the repo history — acceptable trade-off for memory persistence
- If Cloudflare Pages has an outage, blog is down (but GitHub Pages had the same risk)

**What changed:**
- Blog URL: `jimbo.pages.dev` (update bookmarks, RSS readers)
- Internal link pattern: absolute `/jimbo-workspace/blog/...` → relative `./...` or root `/...`
- No feed.xml existed at migration time — now auto-generated by Astro (ADR-027)
